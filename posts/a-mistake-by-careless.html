写代码是个细致活，马虎不得

最近在建设博客的过程中，遇到一个问题，花了很多时间才发现问题的根源所在，但是这个问题完全是由于我的粗心造成的，如果仔细一点，大可不必在这种问题上耽误时间。

事情的来龙去脉是这样的。我想给页面添加一个效果：点击一下鼠标左键就会出现一个心形的图案，片刻之后消失。本来以为会很简单，因为刚刚做过一个类似的功能，拖动鼠标，出现一条像流星一样的尾巴。

首先我用css画了了一个心形的图案，然后用JS把一个函数绑定到点击鼠标的事件上，如果点击的是鼠标左键，就会创建一个div，加上坐标属性，并且调用一这个函数使其在若干毫秒之后消失。

function makeItDisappear(element, milliSecond) { setTimeout(function() { element.remove(); }, milliSecond); }

过程很简单，但是后来发现，remove()这个方法并没有起作用，心形图案显示出来不会自行消失。一开始觉得这个问题很诡异，因为这些代码和实现‘尾巴’效果的代码并没有什么差别。

Google，StackOverFlow，包括jQuery的文档都看了个遍，还是没能找出原因。后来想，干脆把要remove掉的div连同它的parent元素一起remove掉试试，不过要先看一下它的parent是什么，然而答案让我感到很疑惑 - 此element的parent元素是footer，而不是我append到的div.container-fluid，为什么呢？我又去检查了一下HTML，终于发现，卧槽，footer也有container-fluid这个class，调用$('.container-fluid').append()这句代码时，实际上是给两个有'container-fluid'这个类的元素都给加上了这个子元素。remove的时候却只删除了一个，有一个始终是显示的。

好奇心强的人会问，为什么‘尾巴’效果表现得就很正常呢？很简单，因为我append到的是body，而body显然只有一个。

我想说的是，事情往往没那么复杂，写代码时细心一点，虽然感觉上会慢一点，但实际上能为你省去很多调试和修改bug的时间。 Be careful all the time.
