React是如何产生DOM的？
June 2, 2025

  • React

引子#

在我上学那会，还没有前端相关的课程（最接近前端的课程是用前端三剑客——Dreamweaver做HTML网页），学的编程语言是Java或C#。大四去公司实习，一开始也是写Java，后来一个机缘巧合，当时我的mentor让我选是继续做Java还是做前端，当时觉得前端所见即所得，很好玩，于是便开启了我的前端生涯。

带我入门前端领域的第一本书是「JavaScript DOM编程艺术」，当时的前端还方兴未艾，React和Vue还不为大多数人所知，jQuery是当时的主流框架，日常开发就是使用$去查找DOM元素并直接在DOM上进行操作，没经历过那个时代很难想象。

现如今，由于React及虚拟DOM的普及，实际开发中已经很少需要用到查找和直接操作DOM的方法了，这导致我们渐渐遗忘了这些API。本文（可能是一个系列）的初衷就是帮助自己和读者（假如有）搞懂React是如何替我们创建和维护DOM并把背后的复杂性给巧妙隐藏起来的。

正文#

React更新大致分为三个阶段：

  1. Reconciliation 构建新的Fiber Tree，并标记每个节点的变化类型
  2. Commit 根据标记更新实际的DOM，并执行副作用，包括DOM操作、ref、useEffect等
  3. Passive Effects 异步执行useEffect中注册的副作用

在第一阶段，每个Fiber节点会被打上变化标记（存储在flags这个字段上）。 标记类型分为三种；Placement、Update、Deletion，对应新增、更新和删除。

在第二阶段，React会从根节点开始，深度优先遍历Fiber Tree，检查每个节点的标记（flags），然后执行相应的操作。

标记类型 操作
Placement insertDOMNode
Update updateDOMNode
Deletion removeChild

让我们来看一下新增节点的具体实现，这段源码在react/packages/react-reconciler/src/ReactFiberCommitHostEffects.js这个文件中，这里为了便于理解对源码做了一些精简。

 1function commitPlacement(finishedWork: Fiber): void {
 2  let hostParentFiber;
 3  let parentFiber = finishedWork.return;
 4  while (parentFiber !== null) {
 5    if (isHostParent(parentFiber)) {
 6      hostParentFiber = parentFiber;
 7      break;
 8    }
 9    parentFiber = parentFiber.return;
10  }
11
12  if (hostParentFiber == null) {
13    throw new Error(
14      'Expected to find a host parent. This error is likely caused by a bug ' +
15        'in React. Please file an issue.',
16    );
17  }
18
19  switch (hostParentFiber.tag) {
20    case HostSingleton: {
21      if (supportsSingletons) {
22        const parent: Instance = hostParentFiber.stateNode;
23        const before = getHostSibling(finishedWork);
24        // We only have the top Fiber that was inserted but we need to recurse down its
25        // children to find all the terminal nodes.
26        insertOrAppendPlacementNode(
27          finishedWork,
28          before,
29          parent,
30          parentFragmentInstances,
31        );
32        break;
33      }
34      // Fall through
35    }
36    case HostComponent: {
37      const parent: Instance = hostParentFiber.stateNode;
38      if (hostParentFiber.flags & ContentReset) {
39        // Reset the text content of the parent before doing any insertions
40        resetTextContent(parent);
41        // Clear ContentReset from the effect tag
42        hostParentFiber.flags &= ~ContentReset;
43      }
44
45      const before = getHostSibling(finishedWork);
46      // We only have the top Fiber that was inserted but we need to recurse down its
47      // children to find all the terminal nodes.
48      insertOrAppendPlacementNode(
49        finishedWork,
50        before,
51        parent,
52        parentFragmentInstances,
53      );
54      break;
55    }
56    case HostRoot:
57    case HostPortal: {
58      const parent: Container = hostParentFiber.stateNode.containerInfo;
59      const before = getHostSibling(finishedWork);
60      insertOrAppendPlacementNodeIntoContainer(
61        finishedWork,
62        before,
63        parent,
64        parentFragmentInstances,
65      );
66      break;
67    }
68    default:
69      throw new Error(
70        'Invalid host parent fiber. This error is likely caused by a bug ' +
71          'in React. Please file an issue.',
72      );
73  }
74}

fiber.return指向当前fiber的父节点。

向上回溯直到找到最近的宿主节点，如果没找到，直接抛出异常。

调用getHostSibling方法查找当前fiber节点的兄弟DOM节点。

这里有两个关键方法：isHostParent和insertOrAppendPlacementNode，让我们分别看一下这两个方法。

 1function isHostParent(fiber: Fiber): boolean {
 2  return (
 3    fiber.tag === HostComponent ||
 4    fiber.tag === HostRoot ||
 5    (supportsResources ? fiber.tag === HostHoistable : false) ||
 6    (supportsSingletons
 7      ? fiber.tag === HostSingleton && isSingletonScope(fiber.type)
 8      : false) ||
 9    fiber.tag === HostPortal
10  );
11}

该方法用于判断节点是否为合法的宿主节点。 可以看到，tag为HostComponent、HostRoot或HostPortal，就是合法的hostParent。

 1function insertOrAppendPlacementNode(
 2  node: Fiber,
 3  before: ?Instance,
 4  parent: Instance,
 5  parentFragmentInstances: null | Array<FragmentInstanceType>,
 6): void {
 7  const {tag} = node;
 8  const isHost = tag === HostComponent || tag === HostText;
 9  if (isHost) {
10    const stateNode = node.stateNode;
11    if (before) {
12      insertBefore(parent, stateNode, before);
13    } else {
14      appendChild(parent, stateNode);
15    }
16    if (enableFragmentRefs) {
17      commitNewChildToFragmentInstances(node, parentFragmentInstances);
18    }
19    trackHostMutation();
20    return;
21  } else if (tag === HostPortal) {
22    // If the insertion itself is a portal, then we don't want to traverse
23    // down its children. Instead, we'll get insertions from each child in
24    // the portal directly.
25    return;
26  }
27
28  if (
29    (supportsSingletons ? tag === HostSingleton : false) &&
30    isSingletonScope(node.type)
31  ) {
32    // This singleton is the parent of deeper nodes and needs to become
33    // the parent for child insertions and appends
34    parent = node.stateNode;
35  }
36
37  const child = node.child;
38  if (child !== null) {
39    insertOrAppendPlacementNode(child, before, parent, parentFragmentInstances);
40    let sibling = child.sibling;
41    while (sibling !== null) {
42      insertOrAppendPlacementNode(
43        sibling,
44        before,
45        parent,
46        parentFragmentInstances,
47      );
48      sibling = sibling.sibling;
49    }
50  }
51}

在insertOrAppendPlacementNode中，如果有兄弟节点，则调用insertBefore将该节点插入这个兄弟节点前面。没有兄弟节点，再调用appendChild将该节点作为子节点插入父节点中。

为什么要这样做呢。

答案是在不破坏现有DOM的情况下，仅更新必要的部分，并保持DOM的顺序正确。

举个例子

1function App() {
2  const [list, setList] = useState(['A', 'C']);
3  
4  return (
5    <ol>
6      {list.map(item => <li key={item}>{item}</li>)}
7    </ol>
8  )
9}

更新list

setList(['A', 'B', 'C'])

原先的DOM结构为

<li>A</li>
<li>C</li>

新的DOM结构应该为

<li>A</li>
<li>B</li>
<li>C</li>

如果只用appendChild，那么更新后的DOM结构就变为

<li>A</li>
<li>C</li>
<li>B</li>

显然是不正确的。因此要先找到兄弟节点，也就是 <li>C</li>，然后通过insertBefore插在这个节点前面。
